Groups are the foundation of collaboration in Mórchlár. They represent teams, departments, or project collections, and contain projects and members with defined roles.

## Understanding groups

A group in Mórchlár:
- Contains multiple projects
- Has members with different roles (owner, developer, reader)
- Is identified by a unique slug generated from its name
- Serves as the top-level organizational unit

<Note>
Every project must belong to a group. Create groups before creating projects.
</Note>

## Group structure

Groups are stored with the following schema:

```typescript lib/db/schema/groups.ts
export const groups = pgTable("groups", {
    id: serial("id").primaryKey(),
    name: text("name").notNull(),
    slug: text("slug").notNull().unique(),
    createdAt: timestamp("created_at").defaultNow().notNull(),
    updatedAt: timestamp("updated_at")
        .$onUpdate(() => new Date())
        .notNull(),
});
```

### Group members and roles

Members are linked to groups with roles:

```typescript lib/db/schema/groupMembers.ts
export const groupRole = pgEnum('groupRole', ['owner', 'developer', 'reader']);

export const groupMembers = pgTable("groupMembers", {
    groupId: integer("group_id").notNull().references(() => groups.id, { onDelete: 'cascade' }),
    userId: text("user_id").notNull().references(() => user.id, { onDelete: 'cascade' }),
    role: groupRole().notNull(),
}, (table) => [
    primaryKey({ columns: [ table.groupId, table.userId ] }),
]);
```

**Role descriptions**:
- **Owner**: Full control - can manage members, create projects, delete the group
- **Developer**: Can create and manage projects within the group
- **Reader**: Read-only access to group projects

## Creating a group

<Steps>
  <Step title="Navigate to the dashboard">
    After signing in, you'll land on the main dashboard at `/dashboard`. This displays all groups you're a member of.
  </Step>

  <Step title="Click 'Create a group'">
    On the dashboard, you'll see a "Create a group" button alongside existing groups. Click it to start creating a new group.

    <Tip>
    Groups are displayed in a grid layout with four columns, making it easy to see all your teams at a glance.
    </Tip>
  </Step>

  <Step title="Enter group name">
    A prompt will appear asking for the group name. Enter a descriptive name like:
    - "Engineering Team"
    - "Marketing Department"
    - "Q1 2026 Projects"
    - "Client: Acme Corp"

    The name will be used to generate a unique slug for the group URL.
  </Step>

  <Step title="Group is created">
    The group is created immediately and you're assigned as the owner. An alert shows the new group ID, and the group appears in your dashboard.
  </Step>
</Steps>

## Group creation flow (code)

Here's how group creation works:

### Client-side (dashboard page)

```vue app/pages/dashboard/index.vue
async function createGroup() {
    const name = prompt("Enter group name:");
    if (!name) return;

    const id = await groupsStore.createGroup(name);

    alert(`Created group with ID: ${id}`);
}
```

### Server-side (API endpoint)

```typescript server/api/groups/index.post.ts
export default defineAuthenticatedEventHandler(async (event) => {
    const userId = event.context.user.id;
    const bodyData = await validateBody(event, InsertGroup);

    try {
        const createdGroupId = await createGroup(userId, bodyData.name);
        return { id: createdGroupId };
    } catch (error) {
        throw createError({
            statusCode: 500,
            statusMessage: 'Internal Server Error',
        });
    }
})
```

The server:
1. Authenticates the user
2. Validates the group name
3. Generates a unique slug from the name
4. Creates the group record
5. Adds the creator as owner
6. Returns the new group ID

<Warning>
Group slugs must be unique across the entire system. If a slug collision occurs (e.g., two groups named "Team Alpha"), the creation will fail. Consider adding numeric suffixes to handle duplicates.
</Warning>

## Managing group members

Group owners can add members to their groups.

<Steps>
  <Step title="Navigate to group settings">
    From your group's page, go to the members section. This is accessible via the group sidebar.
  </Step>

  <Step title="Enter member username">
    Type the exact GitHub username of the person you want to add:

    ```vue app/pages/dashboard/group/[groupId]/members.vue

        label="Enter the name of the person you would like to add."
    ```

  <Step title="Member is added with role">

    </Note>
  </Step>
</Steps>

### Member addition validation

The server performs extensive validation:

```typescript server/api/groups/[id]/members.post.ts
export default defineAuthenticatedEventHandler(async (event) => {
    const groupId = validateRouterParam(event, 'id');
    const userId = event.context.user.id;
    const body = await validateBody(event, InsertGroupMember);

    const requesteePermissions = await getUserGroupPermissions(userId, groupId);
    
    // Check if group exists and user is in it
    if (!requesteePermissions) {
        throw createError({
            statusCode: 404,
            statusMessage: 'Group not found',
        });
    }

    // Only owners can add members
    if (requesteePermissions.role !== 'owner') {
        throw createError({
            statusCode: 403,
            statusMessage: 'Only owners can add members to groups.',
        });
    }

    // Can't add owners
    if (body.role !== "developer" && body.role !== "reader") {
        throw createError({
            statusCode: 409,
            statusMessage: 'Owners cannot add users with higher or same permissions as them.'
        })
    }

    // Check if user to add exists
    const userToAdd = await getUserByUsername(body.userName);
    if(!userToAdd) {
        throw createError({
            statusCode: 404,
            statusMessage: "User not found",
        });
    }

    // Check if user is already in group
    const toBeAddedInGroup = await getUserGroupPermissions(userToAdd.id, groupId);
    if (toBeAddedInGroup) {
        throw createError({
            statusCode: 409,
            statusMessage: "User is already in this group",
        });
    }

    return await addUserToGroup(userToAdd.id, groupId, body.role);
});
```

**Validation checks**:
1. Group exists and requester is a member
2. Requester is the owner
3. Role being assigned is developer or reader (not owner)
4. User to add exists in the system
5. User is not already a member

<Warning>
Users must have signed in to Mórchlár at least once before they can be added to groups. The system looks them up by their GitHub username.
</Warning>

## Group permissions

Different roles have different capabilities:

| Action | Owner | Developer | Reader |
|--------|-------|-----------|--------|
| View projects | ✓ | ✓ | ✓ |
| Create projects | ✓ | ✓ | ✗ |
| Delete projects | ✓ | ✓ | ✗ |
| Add members | ✓ | ✗ | ✗ |
| Remove members | ✓ | ✗ | ✗ |
| Delete group | ✓ | ✗ | ✗ |

<Note>
Currently, developers have the same project permissions as owners. Fine-grained permissions may be added in future versions.
</Note>

## Best practices

### Naming groups

- Use clear, descriptive names
- Include team or department identifiers
- Consider using prefixes for different types (e.g., "Client: Acme", "Team: Engineering")
- Keep names concise for better UI display

### Organizing groups

- Create separate groups for different clients
- Use groups to separate by team or department
- Consider time-based groups for specific initiatives (e.g., "Q1 2026 Launch")

### Managing members

- Add members as needed rather than preemptively
- Start with developer role and adjust if needed
- Use reader role for stakeholders who need visibility but not edit access
- Document group purpose in your team wiki or documentation

## Limitations and considerations

### Current limitations

- Group names cannot be changed after creation
- Groups cannot be deleted via the UI (database-only operation)
- No group transfer (changing owner)
- No bulk member import

### Database cascade behavior

When a group is deleted:
```typescript
onDelete: 'cascade'
```

- All group members are removed
- All projects in the group are deleted
- All tasks in those projects are deleted

<Warning>
Deleting a group is irreversible and cascades to all related data. Always export important data before deletion.
</Warning>

## Troubleshooting

### "User not found" error

This occurs when adding members who haven't signed into Mórchlár. Solutions:
- Ask the user to sign in once
- Verify the username is their GitHub username, not display name
- Check for typos in the username

### Can't see a group

If you can't see a group you should have access to:
- Verify you've been added as a member
- Check if you're signed in with the correct GitHub account
- Refresh the page to reload group data

### Slug collision errors

If group creation fails due to slug conflicts:
- Try a more unique name
- Add identifying information (e.g., "Engineering 2026" instead of "Engineering")
- Contact an admin to resolve conflicts
