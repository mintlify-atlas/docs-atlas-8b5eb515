# Quick start

This guide will help you set up Mórchlár locally and get your first group and project running in minutes.

<Note>
  Before you begin, ensure you have:
  - **Bun** installed (v1.0+)
  - **PostgreSQL** database (we recommend Neon for serverless PostgreSQL)
  - **GitHub OAuth app** credentials ([create one here](https://github.com/settings/developers))
</Note>

## Setup steps

<Steps>
  <Step title="Clone the repository">
    Clone the Mórchlár repository to your local machine:

    ```bash
    git clone <repository-url>
    cd team-project-ca
    ```
  </Step>

  <Step title="Install dependencies">
    Use Bun to install all required packages:

    ```bash
    bun install
    ```

    This will install all dependencies including Nuxt 3, Vue 3, Drizzle ORM, better-auth, and more.
  </Step>

  <Step title="Configure environment variables">
    Create a `.env` file in the root directory by copying the example:

    ```bash
    cp .env.example .env
    ```

    Then fill in your environment variables:

    ```bash
    # Database connection string (Neon, Railway, local PostgreSQL, etc.)
    DATABASE_URL=postgresql://user:password@host:5432/database

    # Authentication secret (generate a random string)
    BETTER_AUTH_SECRET=your-super-secret-key-here

    # Application URL
    BETTER_AUTH_URL=http://localhost:3000

    # GitHub OAuth credentials
    GITHUB_CLIENT_ID=your-github-client-id
    GITHUB_CLIENT_SECRET=your-github-client-secret
    ```

    <Warning>
      For `BETTER_AUTH_SECRET`, use a strong random string. You can generate one with:
      ```bash
      openssl rand -base64 32
      ```
    </Warning>

    ### GitHub OAuth setup

    To get your GitHub OAuth credentials:

    1. Go to [GitHub Developer Settings](https://github.com/settings/developers)
    2. Click "New OAuth App"
    3. Fill in the details:
       - **Application name**: Mórchlár (Local)
       - **Homepage URL**: `http://localhost:3000`
       - **Authorization callback URL**: `http://localhost:3000/api/auth/callback/github`
    4. Copy the Client ID and generate a Client Secret
    5. Add them to your `.env` file

    <Note>
      Mórchlár requests two GitHub scopes:
      - `user:email`: Access your email address
      - `repo`: Access your repositories (needed to verify repo access and integration)
    </Note>
  </Step>

  <Step title="Run database migrations">
    Generate and run the database migrations to set up your schema:

    ```bash
    # Generate migration files (if needed)
    bun run db:generate

    # Apply migrations to your database
    bun run db:migrate
    ```

    This creates all necessary tables:
    - `groups` - Team organizations
    - `groupMembers` - User memberships with roles
    - `projects` - GitHub-linked projects
    - `tasks` - Timeline-based tasks
    - `user`, `session`, `account`, `verification` - Auth tables

    The migration system uses Drizzle Kit with the configuration in `drizzle.config.ts`:

    ```typescript
    export default defineConfig({
        out: './lib/db/migrations',
        schema: './lib/db/schema',
        casing: 'snake_case',
        dialect: 'postgresql',
        dbCredentials: {
            url: env.DATABASE_URL,
        },
    });
    ```
  </Step>

  <Step title="Start the development server">
    Launch the Nuxt development server:

    ```bash
    bun run dev
    ```

    The application will start at `http://localhost:3000`. You should see output like:

    ```
    Nuxt 4.3.1 with Nitro 2.10.7
    
      ➜ Local:    http://localhost:3000/
      ➜ Network:  use --host to expose
    ```
  </Step>

  <Step title="Sign in with GitHub">
    1. Open your browser and navigate to `http://localhost:3000`
    2. Click the "Sign in with GitHub" button
    3. Authorize the application when prompted by GitHub
    4. You'll be redirected to the dashboard at `/dashboard`

    The authentication flow uses better-auth with the GitHub provider:

    ```typescript
    // lib/auth.ts
    export const auth = betterAuth({
        database: drizzleAdapter(db, {
            provider: 'pg',
            schema,
        }),
        socialProviders: {
            github: {
                clientId: env.GITHUB_CLIENT_ID,
                clientSecret: env.GITHUB_CLIENT_SECRET,
                scope: [ 'user:email', 'repo' ]
            },
        },
    });
    ```
  </Step>

  <Step title="Create your first group">
    On the dashboard:

    1. Click "Create a group"
    2. Enter a name for your team (e.g., "My Team")
    3. The group is created with you as the owner

    Behind the scenes, this calls the groups API:

    ```typescript
    // server/api/groups/index.post.ts
    export default defineAuthenticatedEventHandler(async (event) => {
        const userId = event.context.user.id;
        const bodyData = await validateBody(event, InsertGroup);
        
        const createdGroupId = await createGroup(userId, bodyData.name);
        
        return { id: createdGroupId };
    });
    ```

    The `createGroup` function:
    - Generates a unique slug from the group name
    - Inserts the group record
    - Adds you as a member with the "owner" role

    ```typescript
    // lib/db/queries/groups.ts
    export async function createGroup(userId: string, groupName: string) {
        const inserted = await db.insert(groups).values({
            name: groupName,
            slug: await findUniqueSlug(groupName),
        }).returning();

        const result = inserted[0];
        if (!result) throw Error("Error creating group");

        await db.insert(groupMembers).values({
            groupId: result.id,
            userId: userId,
            role: 'owner',
        });

        return result.id;
    }
    ```
  </Step>

  <Step title="Add a project">
    Within your group:

    1. Click into your newly created group
    2. Create a new project
    3. Enter a project title
    4. Select a GitHub repository you have access to
    5. The project is created and linked to your repo

    The project creation validates GitHub repository access:

    ```typescript
    // server/api/projects/index.post.ts
    export default defineAuthenticatedEventHandler(async (event) => {
        const userId = event.context.user.id;
        const bodyData = await validateBody(event, ClientInsertProject);
        
        await ensureUserInGroup(userId, bodyData.groupId);
        
        // Validate GitHub repo access
        const token = await getUserGitHubAuthToken(userId);
        const repoStatus = await verifyGitHubRepoAccess(token, bodyData.repo);
        
        if (!repoStatus.valid) {
            throw createError({
                statusCode: 400,
                message: 'Invalid repo'
            });
        }
        
        const createdProjectId = await createProject(
            repoStatus.id,
            repoStatus.name,
            repoStatus.owner,
            bodyData.title,
            bodyData.groupId
        );
        
        return { id: createdProjectId };
    });
    ```

    <Note>
      The repository validation ensures:
      - You have access to the repo
      - The repo exists on GitHub
      - Your OAuth token has the necessary permissions
    </Note>
  </Step>
</Steps>

## Development workflow

### Database schema changes

When you modify the database schema:

```bash
# 1. Update schema files in lib/db/schema/
# 2. Generate migration
bun run db:generate

# 3. Review the generated SQL in lib/db/migrations/
# 4. Apply migration
bun run db:migrate
```

### Available scripts

```json
{
  "dev": "nuxt dev",              // Start dev server
  "build": "nuxt build",          // Build for production
  "preview": "nuxt preview",      // Preview production build
  "db:generate": "bun drizzle-kit generate",  // Generate migrations
  "db:migrate": "bun drizzle-kit migrate"     // Run migrations
}
```

## Troubleshooting

### Database connection issues

If you see connection errors:

1. Verify your `DATABASE_URL` is correct
2. Ensure your PostgreSQL instance is running
3. Check that your IP is allowed (for cloud databases like Neon)
4. Test the connection string manually

### GitHub OAuth errors

If authentication fails:

1. Verify your callback URL matches: `http://localhost:3000/api/auth/callback/github`
2. Check that Client ID and Secret are correct
3. Ensure the OAuth app is active on GitHub
4. Clear your browser cookies and try again

### Migration errors

If migrations fail:

```bash
# Check current migration status
bun drizzle-kit check

# Drop and recreate (⚠️ destroys data)
bun drizzle-kit drop
bun run db:migrate
```

<Warning>
  The environment validation in `lib/env.ts` will throw clear errors if required variables are missing. Make sure all values in `.env.example` are filled in your `.env` file.
</Warning>
