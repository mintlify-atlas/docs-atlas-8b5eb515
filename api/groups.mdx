## Overview

Groups represent teams or organizations in Mórchlár. Each group can contain multiple members with different roles, and projects belong to groups.

## Group roles

Groups use a role-based permission system:

- **owner** - Full control over the group, can add/remove members and manage projects
- **developer** - Can create and manage projects within the group
- **reader** - Read-only access to group projects

## Create a group

<CodeGroup>
```bash cURL
curl -X POST https://your-domain.com/api/groups \
  -H "Content-Type: application/json" \
  -H "Cookie: better_auth.session_token=your-token" \
  -d '{
    "name": "My Team"
  }'
```

```json Response
{
  "id": 1
}
```
</CodeGroup>

Creates a new group with the authenticated user as the owner. A URL-friendly slug is automatically generated from the group name.

### Request body

<ParamField body="name" type="string" required>
  The name of the group. Must be unique across the platform.
</ParamField>

### Response

<ResponseField name="id" type="number">
  The unique identifier for the created group.
</ResponseField>

### Errors

- `401 Unauthorized` - User is not authenticated
- `500 Internal Server Error` - Database error or duplicate group name

## Add member to group

<CodeGroup>
```bash cURL
curl -X POST https://your-domain.com/api/groups/1/members \
  -H "Content-Type: application/json" \
  -H "Cookie: better_auth.session_token=your-token" \
  -d '{
    "userName": "johndoe",
    "role": "developer"
  }'
```

```json Response
{
  "groupId": 1,
  "userId": "user_abc123",
  "role": "developer"
}
```
</CodeGroup>

Adds a user to a group with the specified role. Only group owners can add members.

### Path parameters

<ParamField path="id" type="number" required>
  The unique identifier of the group.
</ParamField>

### Request body

<ParamField body="userName" type="string" required>
  The username of the user to add to the group.
</ParamField>

<ParamField body="role" type="string" required>
  The role to assign: `developer` or `reader`. Owners cannot add users with the `owner` role.
</ParamField>

### Response

<ResponseField name="groupId" type="number">
  The group ID.
</ResponseField>

<ResponseField name="userId" type="string">
  The user ID of the added member.
</ResponseField>

<ResponseField name="role" type="string">
  The role assigned to the member.
</ResponseField>

### Errors

- `401 Unauthorized` - User is not authenticated
- `403 Forbidden` - Only owners can add members to groups
- `404 Not Found` - Group or user not found
- `409 Conflict` - User is already a member of the group, or attempting to assign owner role

## Schema reference

Groups are stored with the following schema:

```typescript
{
  id: number;              // Auto-generated primary key
  name: string;            // Group name (required)
  slug: string;            // URL-friendly identifier (auto-generated, unique)
  createdAt: Date;         // Creation timestamp
  updatedAt: Date;         // Last update timestamp
}
```

Group memberships link users to groups:

```typescript
{
  groupId: number;         // Foreign key to groups table
  userId: string;          // Foreign key to user table
  role: 'owner' | 'developer' | 'reader';  // Member role
}
```
